# Data Lab 团队内部速查

本文面向团队内部的管理员和复审人员，重点说明日常操作。标注员请使用单独的录屏教程。

## 一、启动平台

双击项目根目录的 `启动平台.cmd`：

- 选 `2. 本机运行`：需要只在本机使用时启动完整 Next.js 平台，默认地址为 `http://127.0.0.1:3000`。
- 选 `3. 公网审核（ngrok）`：先启动或复用本机平台，再建立 ngrok HTTPS 公网地址，供外部标注员访问。
- 选 `4. 查看状态`：查看本机服务和当前 ngrok 地址。
- 选 `5. 备份数据库`：备份 SQLite 数据库。
- 选 `6. 停止服务`：停止本启动器负责启动的 Next.js/ngrok 进程。

首次在一台新电脑部署时选 `1. 首次初始化`。它会创建 `.env`、安装缺失依赖，并在没有数据库时迁移和初始化数据。若已经存在 `prisma/dev.db`，为了避免误改真实数据，启动器不会自动执行新迁移；升级代码后应先备份，再由维护人员执行 `npm run db:migrate`。

这个启动器覆盖同一个 Next.js 应用里的首页、教学端和整个 Data Lab（账号、任务、标注、审核、发布、训练登记、评测登记）。SQLite 不需要单独启动。它不负责启动外部大模型服务、Prisma Studio、训练平台任务或双盲评测脚本；公网模式还要求电脑已安装并配置 ngrok。

## 二、创建账号

进入“高级管理 → 后台账号”。

1. 填写用户名、显示名、初始密码和身份，创建账号。
2. 把初始密码私下交给本人，不要放进公共群聊或教程视频。
3. 已有账号可编辑显示名、用户名和身份，也可重置密码或停用。
4. 有历史任务或审核记录的账号不能永久删除，应使用“停用”；没有任何业务记录的空账号才允许删除。

停用账号会立即失去访问权限，也不会进入新的任务分配名单。

## 三、下发标注任务

进入“任务分配 → 创建任务分配”。

1. 填任务名称，例如“七月第一批人工标注”。
2. 选择数据批次；“最多处理多少条样本”填 `0` 表示全部。
3. 勾选参与人员，并为每人填写“最多领取”。填 `0` 表示该参与者不限条数。
4. 选择审核强度。正式批量工作建议选“每条由 2 人独立标注”；规则稳定且希望提高速度时才选单人标注。
5. 一般不需要展开高级选项；只有需要限制阶段、候选等级或回复风格时再调整。
6. 点击“创建任务分配”，然后在下方活动卡片点击“启动”。草稿未启动时，标注员领不到任务。

活动卡片中：

- “有效任务”是已通过工作量审核的任务数；双标样本的两位标注员各算 1 条。
- “已提交任务”包含尚未审核的提交。
- “待工作量审核”必须由管理员处理。
- “待仲裁”表示两个有效版本仍需最终选版。

如果标注员说没有任务，先检查活动是否启动、本人是否被选为参与者、个人领取上限是否已用完。若页面提示“双标任务需要其他标注员完成”，这是正常的双盲隔离，不要把同一样本再次发给本人。

## 四、确认有效工作量

进入“有效标注统计”。

1. 在“逐条审核”中展开“查看本次提交内容”。
2. 内容合格时点击“通过并计 1 条”。
3. 可以修正时，填写审核说明后点“退回修改”；任务会回到原标注员。
4. 提交无效、需要别人重做时，填写说明并点“标记无效”；该任务重新开放，原提交不计数。
5. 结算条数只看每个人的“审核通过/有效条数”，不要按已分配或已提交统计。
6. 需要留档时点击“导出逐条明细 CSV”。后台只记录条数，不记录单价或金额。

工作量审核与数据最终选版是两件事：双标的两个合格提交都可以各计 1 条，但数据集中最终只保留仲裁选中的版本。

## 五、数据仲裁

进入“数据仲裁”。系统对不同标注员的版本进行匿名展示。

1. 比较两个版本是否符合当前阶段目标及结构要求。
2. 选择更好的版本，或按页面提供的方式合并/定级。
3. 两个版本都不合格时退回或拒绝，不要为了清空队列勉强选一个。

只有相关提交先通过“有效标注统计”的工作量审核后，双标样本才会进入可仲裁状态。

## 六、下载完全标注好的数据集

先在“任务分配”确认目标活动满足以下条件：

- “有效任务”为总任务数，即 `有效任务 = 全部任务`；
- “待工作量审核”为 `0`；
- “待仲裁”为 `0`；
- “完成样本”为目标样本总数。

然后进入“高级管理 → 数据版本”：

1. 填写唯一版本号，例如 `stem-sft-2026-07-v1`。
2. 选择刚刚确认完成的标注活动。
3. 一般保持 Gold 权重 `1.5`、Silver 权重 `1.0`，点击“创建版本草稿”。
4. 再次核对活动后，点击“冻结发布”。冻结后不可修改。
5. 冻结成功后下载：
   - `clean`：Gold 与 Reviewed Silver 合并后的完整可训练 ShareGPT 数据，通常下载这个；
   - `gold`：只含人工仲裁后的 Human Gold；
   - `silver`：只含工作量审核通过的 Reviewed Silver；
   - `manifest`：版本、数量、阶段分布、样本来源和权重清单，用于留档和复现。

重要：当前系统只要已有合格候选就允许冻结，不会替团队判断活动是否已经 100% 完成。因此必须先按上面的四项条件人工确认，不能提前冻结。

## 七、填写训练登记

这里是外部训练平台的留档页，不会自动提交训练，也不保存平台密钥。先在主办方或云平台创建训练任务，再进入“高级管理 → 训练登记”。

示例：

| 字段 | 示例 | 怎么填 |
|---|---|---|
| 任务名称 | 七月第一轮 SFT | 团队内部可识别的名称 |
| 数据版本 | `stem-sft-2026-07-v1` | 只能选择已经冻结的版本 |
| 基础模型 | `Qwen3.5-35B-A3B` | 外部训练实际使用的底模 |
| 主办方任务 ID | `task-20260713-001` | 外部平台返回的任务号；尚未提交可留空 |
| 模型标签 | `qwen-stem-sft-v1` | 后续双盲采集所使用的稳定标签 |
| 状态 | `RUNNING` | `DRAFT` 未提交、`SUBMITTED` 已提交、`RUNNING` 运行中、`SUCCEEDED` 成功、`FAILED` 失败 |
| 训练参数 JSON | `{"learning_rate":0.00002,"epochs":3}` | 必须是合法 JSON；不知道时填 `{}` |
| 备注 | 使用 clean 文件；平台默认模板 | 记录数据文件、特殊配置或失败原因 |

登记前核对数据版本、基础模型和外部任务 ID，避免用同名记录混淆不同轮训练。

## 八、填写双盲结果

双盲采集和裁判由命令行脚本完成，网页只负责保存结果和汇总。维护人员运行：

```powershell
npx tsx scripts/blind-eval.ts collect --scope smoke --tag <模型标签>
npx tsx scripts/blind-eval.ts judge <模型标签A> <模型标签B> --scope smoke
```

脚本产物位于 `data/blind-eval/`，常见文件为：

- `transcript-<模型标签>.json`：某个模型的完整对话采集结果；
- `verdict-<模型A>-vs-<模型B>.json`：匿名裁判结果。

进入“高级管理 → 双盲评测”：

1. 填评测名称，例如 `qwen-stem-v1-vs-baseline-smoke`。
2. 一次选择两个模型对应的 transcript 文件和同一组 verdict 文件；文件必须是脚本原始 JSON，并包含 `schemaVersion`。
3. 点击“导入评测”。
4. 检查页面显示的模型 A/B 标签、scope、产物数量，以及 A 胜、B 胜、平局和不一致数量是否与命令行输出一致。

不要手工交换 A/B 标签或改写 verdict JSON；网页会按文件内的标签读取。首次只跑 `smoke` 检查流程，确认模型、裁判和产物都正常后再运行完整范围。
