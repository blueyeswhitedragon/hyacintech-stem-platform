# 模型迭代闭环设计

详细数据模型、API、页面、迁移和测试方案见 [`model-improvement-loop-implementation-design.md`](./model-improvement-loop-implementation-design.md)。

## 目标

平台需要形成两条真正可追踪、可回滚的闭环：

1. **可控回复风格闭环**：任务分配中的风格目标必须进入标注规范、训练样本、在线提示词和分风格评测，而不是只显示在标注页面。
2. **生产数据安全回流闭环**：真实教学会话只能作为待发现问题的原始证据，不能自动成为模型自己的 SFT 正样本。

## 不可绕过的原则

- 线上模型原始回答默认是 **隔离候选**，不是训练标签。
- 学生完成阶段、教师通过作业或对话顺利结束，只是候选筛选信号，不等于导师回答正确。
- 原模型回答未经独立人工纠正和复核，不得进入该模型后继版本的 SFT 正样本。
- 人工纠正后的回答可作为 SFT 正样本；原回答与纠正回答可组成偏好数据的 rejected/chosen 对。
- “无需修改”的线上回答至少需要独立人工复核，并进入较低置信等级；不能仅凭模型自评或自动评分升级为 Gold。
- 训练集、盲测集和回归集按稳定指纹隔离，任何训练版本不得包含对应评测样本或其近重复版本。
- 每条回流记录必须保留来源模型、提示词版本、生成参数、请求风格、父数据版本和人工处理链路。
- 涉及学生的真实数据默认不回流；只有在明确授权、最小化采集和脱敏后才能进入人工候选池。图片与附件默认排除。
- Guest/直接体验数据继续不落库，也不参与回流。

## 生产数据状态机

```text
正式会话
  → 授权与范围检查
  → 隔离候选 RAW_QUARANTINED
  → 脱敏与近重复检查 REDACTED
  → 管理员提名 NOMINATED
  → 标注员独立纠正 HUMAN_CORRECTED
  → 工作量审核 + 匿名仲裁 APPROVED
  → 冻结数据版本 RELEASED
  → 训练登记 TRAINED
  → 双盲/回归评测 EVALUATED
  → 灰度部署 PROMOTED
  → 新一轮线上观测
```

任一步都可以进入 `REJECTED`，拒绝原因必须留档。原始隔离数据不直接出现在训练下载文件中。

## 哪些数据可以训练

| 数据 | 用途 | 是否可直接进入 SFT |
|---|---|---|
| 线上模型原始回答 | 发现失败、构造偏好负例 | 否 |
| 人工实质纠正且独立复核的回答 | SFT 正样本 | 是 |
| 原回答 + 人工纠正回答 | DPO/偏好训练候选 | 可以，但需独立复核 |
| 人工确认“无需修改”的原回答 | Reviewed Silver 候选 | 默认不进入 Gold，需独立复核 |
| 教师评分、阶段通过、学生完成率 | 采样和优先级信号 | 否 |
| 自动裁判或模型自评结果 | 辅助排序、发现异常 | 否 |
| 学生输入 | 经授权与脱敏后作为上下文 | 不能单独成为导师正标签 |

## 必须记录的血缘

每条生产候选至少记录：

- `conversationId`、阶段、轮次与稳定内容指纹；
- `sourceModelTag`、`sourceTrainingRunId`、`sourceReleaseId`；
- `promptVersion`、结构契约版本、生成参数；
- `requestedStyleFamily` 与实际在线风格指令版本；
- 授权范围、脱敏状态、附件排除状态；
- 提名人、纠正人、工作量审核人、仲裁人和时间；
- `transformationType`：实质纠正、轻微编辑、无需修改、拒绝；
- 最终进入的数据版本、训练任务和部署版本。

模型注册表需要明确 `parentModelVersionId`。平台在创建训练版本时执行来源策略检查，阻止将某模型未经人工变换的原始回答用于其自身后继版本。

## 风格闭环

当前五种风格继续使用稳定标识：

- `socratic_concise`：苏格拉底简洁型；
- `warm_companion`：温和陪伴型；
- `engineering_mentor`：工程导师型；
- `evidence_analyst`：证据分析型；
- `classroom_coach`：课堂教练型。

闭环要求：

1. 每种风格拥有可执行规范：语气、单轮长度、追问数量、证据要求、允许与禁止行为。
2. Data Lab 分配的 `styleFamily` 写入修订和冻结版本的元数据，并在 manifest 汇总实际入选比例。
3. 微调输入中加入模型可见的稳定风格指令；不能只把风格放在训练框架可能忽略的 metadata。
4. 教师在发布作业时选择默认导师风格，也可选择“自动适配”；选择结果固化到作业和会话，不能随请求漂移。
5. 正式聊天通过统一的 `getPromptForPhase()` 注入相同版本的风格指令。
6. 双盲评测按阶段和风格分别汇总，防止整体平均分掩盖某一种风格的退化。
7. 生产回流记录保留请求风格，人工复核时同时判断内容质量和风格遵循度。

## 产品入口

### 教师端

- 发布作业时选择导师风格；默认“自动适配”。
- 明确选择该作业是否允许将脱敏后的对话提名为改进候选。
- 教师可以提名具体失败轮次并填写问题说明，但不能直接把它标记为训练正样本。

### Data Lab 管理员端

- 新增“线上候选”页：查看授权范围、来源模型、阶段、失败信号和脱敏结果。
- 管理员只能提名或拒绝；进入现有标注系统后，仍按工作量审核和匿名仲裁执行。
- 候选导入后使用新的不可变批次，`sourceType` 标记为 `production_trace`。

### 模型管理端

- 新增模型版本与部署记录，关联训练任务、数据版本、父模型、提示词版本和评测结果。
- 只有通过规定盲测和回归阈值的模型版本才允许标记为可部署。
- 部署保留当前版本和回滚目标，不直接覆盖历史记录。

## 分阶段实施

### M9A：风格可控闭环

- 建立五种风格规范与版本号。
- 作业/会话持久化风格选择。
- 在线提示词消费风格。
- 标注修订、发布记录和 manifest 保留风格。
- 增加分风格单测与导出检查。

### M9B：生产候选隔离与提名

- 增加生产候选、授权、脱敏和来源血缘模型。
- 只允许正式账号的已完成会话进入隔离候选；Guest 永不进入。
- 提供管理员候选页和教师提名入口。
- 实现附件默认排除、文本脱敏、去重和评测集污染检查。

### M9C：人工纠正回流

- 将合格候选转换为不可变 Data Lab 批次。
- 复用双人标注、有效工作量审核和匿名仲裁。
- 增加实质纠正强度和训练资格校验。
- 导出 SFT 与偏好数据时分别执行资格策略。

### M9D：模型血缘、评测与灰度部署

- 建立模型版本、父版本、训练数据和提示词血缘。
- 训练前阻止自蒸馏污染与评测集泄漏。
- 双盲、结构契约、阶段推进和分风格回归共同作为晋级门槛。
- 登记灰度部署、监控窗口、回滚目标和最终晋级决策。

## 验收标准

- 在线选择任一风格后，服务端保存选择并在每轮提示词中使用同一风格规范版本。
- 冻结数据 manifest 能准确显示每种风格的实际入选数量，而不是任务分配数量。
- 任何模型原始线上回答都不能绕过人工纠正/复核成为其后继模型的 SFT 正样本。
- 可从一个训练样本追溯到原会话、来源模型、人工纠正、仲裁、数据版本、训练任务和部署版本。
- 训练创建前自动检查模型血缘、重复样本和评测集污染，并提供可审计的阻止原因。
- 新模型只有通过基线对照、分阶段和分风格评测后才能进入灰度部署。
